# 📊 Gold Layer Data Catalog

> **Overview**
> The Gold Layer serves as the finalized, business-ready data representation. It is optimized for enterprise reporting, dashboarding, and advanced analytical modeling. The schema follows a dimensional modeling approach, separating business entities into **Dimensions** and transactional events into **Facts**.

---

## 🧑‍🤝‍🧑 1. `gold.dim_customers`
**Purpose:** Maintains enriched customer profiles, integrating demographic and geographic dimensions to support customer segmentation and behavior analysis.

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| `customer_key` | **INT** | Auto-generated surrogate primary key for distinct customer record identification. |
| `customer_id` | INT | Source system's native numeric identifier for the customer. |
| `customer_number` | NVARCHAR(50) | Business-facing alphanumeric code used for external tracking and CRM matching. |
| `first_name` | NVARCHAR(50) | Customer's given name. |
| `last_name` | NVARCHAR(50) | Customer's surname or family name. |
| `country` | NVARCHAR(50) | Geographic country of residence. |
| `marital_status` | NVARCHAR(50) | Current marital status indicator (e.g., Single, Married). |
| `gender` | NVARCHAR(50) | Customer gender classification. |
| `birthdate` | DATE | Date of birth, structured in YYYY-MM-DD format. |
| `create_date` | DATE | Timestamp indicating when the customer profile was first onboarded into the source system. |

---

## 📦 2. `gold.dim_products`
**Purpose:** Defines the product catalog, detailing hierarchies, classifications, and historical attributes for inventory and sales performance tracking.

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| `product_key` | **INT** | Surrogate primary key uniquely identifying the product dimension record. |
| `product_id` | INT | Native source system identifier for the item. |
| `product_number` | NVARCHAR(50) | SKU or standard business alphanumeric code for inventory control. |
| `product_name` | NVARCHAR(50) | Full display name of the item, including standard variants like color or size. |
| `category_id` | NVARCHAR(50) | System ID mapping the item to its high-level product category. |
| `category` | NVARCHAR(50) | Macro-level product grouping (e.g., Bikes, Accessories). |
| `subcategory` | NVARCHAR(50) | Micro-level product grouping for granular filtering and reporting. |
| `maintenance_required` | NVARCHAR(50) | Flag denoting if the item is subject to ongoing maintenance schedules. |
| `cost` | INT | Standard base unit cost of the item. |
| `product_line` | NVARCHAR(50) | Associated product line or overarching family grouping. |
| `start_date` | DATE | Official launch or availability start date for the product lifecycle. |

---

## 📈 3. `gold.fact_sales`
**Purpose:** Captures quantifiable transactional events. This table forms the core of the star schema, tracking revenue, volume, and order fulfillment timelines.

| Column Name | Data Type | Description |
| :--- | :--- | :--- |
| `order_number` | NVARCHAR(50) | Transaction identifier or official invoice number. |
| `product_key` | **INT** | Foreign key linking the transaction line to `gold.dim_products`. |
| `customer_key` | **INT** | Foreign key linking the transaction to `gold.dim_customers`. |
| `order_date` | DATE | Date the transaction was formally initiated. |
| `shipping_date` | DATE | Date the physical goods were dispatched for delivery. |
| `due_date` | DATE | Expected payment settlement or fulfillment deadline. |
| `sales_amount` | INT | Total gross revenue generated by this specific line item. |
| `quantity` | INT | Count of individual units purchased in this transaction line. |
| `price` | INT | Executed selling price per individual unit. |
